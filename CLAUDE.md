## Project

This project is a **software company website**. The information architecture below was
synthesized from two reference sites:

- [themobilereality.com](https://themobilereality.com) — a conversion-focused, long-scroll
  agency site (sticky multi-dropdown nav, value-prop hero with dual CTAs, alternating
  proof/offer sections, repeated CTA banners, content-rich footer).
- [leewayhertz.com](https://www.leewayhertz.com) — a solution-centric site with broad,
  multi-axis navigation (by technology, by industry, by engagement model).

Use this document as the canonical reference for the site's structure, page types, and
navigation when scaffolding pages, building components, or writing content.

## Conventions & Principles

- **Conversion-first**: every primary page should surface a clear CTA. Reuse the global
  CTA banner between major sections on long pages.
- **Multi-axis navigation**: offerings are discoverable by technology, by industry, and by
  engagement model. Keep these three axes consistent across nav, footer, and landing pages.
- **Proof alternates with offer**: interleave credibility sections (case studies, logos,
  testimonials, awards) with capability sections (services, expertise, tech stack).
- **Landing-page template**: service, industry, and technology pages share a common
  template — overview → details → process → related case studies → CTA.
